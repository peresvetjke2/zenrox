# UC-007: Open Tasks Retrieval

## Goal

Пользователь задает вопрос о своих открытых делах и получает полный user-visible список задач в состоянии `open`, включая задачи без срока выполнения.

## Primary Actor

Автор проекта.

## Trigger

Пользователь отправляет канонический вопрос `какие у меня открытые дела?` или его тривиальный текстовый вариант без смены смысла через основной conversational интерфейс.

## Preconditions

- В системе уже могут существовать задачи со статусами `open` и `done`.
- Система умеет распознать запрос как retrieval-intent первого MVP, а не как capture-команду.
- Источник истины по статусу задачи находится в owner-слое task storage.

## Main Flow

1. Пользователь отправляет вопрос об открытых задачах.
2. Система распознает вопрос как запрос полного списка открытых дел первого MVP.
3. Система читает задачи из personal memory и выбирает только записи со статусом `open`.
4. Система формирует понятный текстовый ответ со списком найденных задач или явным empty-state ответом.
5. Пользователь получает ответ без изменения сохраненных задач.

## Alternate Flows / Exceptions

- `ALT-01` Если открытых задач нет, система явно сообщает, что сейчас открытых дел не найдено.
- `ALT-02` Если среди открытых задач есть записи без срока выполнения, они все равно включаются в ответ.
- `EX-01` Если запрос похож на retrieval, но выходит за узкий scope первого MVP, система явно сообщает, что сейчас поддерживается только вопрос про полный список открытых дел, и не сохраняет запрос как новую задачу.
- `EX-02` Если чтение task storage завершается неуспешно, система сообщает, что не смогла получить список открытых дел, и не выдает частичный или выдуманный ответ.

## Postconditions

- После успешного завершения пользователь получает полный список текущих `open`-задач или явный empty-state ответ.
- Ни одна задача не создается, не обновляется и не переводится в другой статус только из-за retrieval-запроса.
- После неуспешного завершения пользователь понимает, что reliable ответ не был получен.

## Business Rules

- `BR-01` Для первого MVP `open task` означает любую задачу, которая еще не находится в статусе `done`.
- `BR-02` Retrieval открытых дел не зависит от наличия даты, проекта, категории или других дополнительных атрибутов.
- `BR-03` В первом MVP вопрос про открытые дела означает канонический запрос `какие у меня открытые дела?` и его тривиальные варианты по регистру и пунктуации; этот intent не должен маршрутизироваться в capture-path.
- `BR-04` Если retrieval-вопрос выходит за scope первого MVP, система обязана вернуть явный unsupported verdict с указанием, что первый retrieval slice поддерживает только вопрос про полный список открытых дел, вместо guessed answer и вместо автосохранения запроса как задачи.
- `BR-05` Read-only retrieval не должен менять сохраненные данные или создавать новые task records.
