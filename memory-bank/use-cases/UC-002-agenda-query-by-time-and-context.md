# UC-002: Agenda Query By Time And Context

## Goal

Пользователь задает свободный вопрос о текущих делах, а система возвращает релевантный ответ по уже сохраненным задачам с учетом временного диапазона и, когда возможно, пользовательского контекста.

## Primary Actor

Автор проекта.

## Trigger

Пользователь задает вопрос вроде "есть ли у меня дела на эту неделю?" или "что мне нужно сделать по работе сегодня?".

## Preconditions

- В системе уже сохранены задачи, которые могут быть релевантны запросу.
- Система умеет извлечь или интерпретировать временной диапазон запроса.
- Если в запросе присутствует контекст вроде "по работе", система либо распознает его, либо умеет запросить уточнение.

## Main Flow

1. Пользователь отправляет свободный вопрос по текущим задачам.
2. Система определяет, какой временной срез, статус или контекст требуется для ответа.
3. Система поднимает релевантные задачи из personal memory.
4. Система формирует понятный ответ в conversational форме.
5. Пользователь получает список, summary или прямой ответ по найденным задачам.

## Alternate Flows / Exceptions

- `ALT-01` Если вопрос не содержит явного временного диапазона, система может использовать естественную интерпретацию формулировки или запросить уточнение.
- `ALT-02` Если контекст вроде "по работе" распознан не полностью, система предпочитает сначала уточнить фильтр, а не смешивать в одном ответе несколько возможных контекстов.
- `EX-01` Если подходящих задач нет, система честно сообщает, что по текущим данным релевантных задач не найдено.
- `EX-02` Если запрос сформулирован слишком неоднозначно, система не придумывает ответ и уходит в clarification.

## Postconditions

- После успешного завершения пользователь получает релевантный ответ по существующим задачам.
- Никакие пользовательские данные не должны быть изменены только из-за факта чтения и ответа на query.
- После неуспешного завершения пользователь должен понимать, что система не смогла надежно определить нужный срез.

## Business Rules

- `BR-01` Вопрос по задачам может формулироваться в свободной форме.
- `BR-02` Система должна по возможности сама выводить временной и контекстный фильтр из запроса.
- `BR-03` При недостаточной уверенности в интерпретации фильтра система предпочитает clarification вместо guessed answer.
- `BR-04` Если ответ можно построить напрямую по структурированным task data, этот путь предпочтительнее лишнего LLM reasoning.
